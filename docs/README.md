# OctoAcme — プロジェクト管理プロセス概要

OctoAcme のプロジェクト管理プロセスは、イニシエーションからプランニング、実行、リリース、振り返りに至る一連のライフサイクルに沿って設計されています。プロジェクトは最初にワンページのプロジェクト・チャーターで目的と成功指標を定義し、キックオフを経て優先化されたバックログと「完了の定義（DoD）」を固めます。計画段階ではリスクや依存関係を明示し、スプリントやリリースに向けたマイルストーンを作成します。

日々の実行ではプロジェクトボード（例：GitHub Projects）のカラム（Backlog → Ready → In Progress → In Review → QA → Done）を使い、小さなプルリクエスト（目安 400 行以内）と CI を活用した PR ワークフローで進めます。定期的なリズムとしてデイリースタンドアップ、週次デリバリー同期、スプリント末のデモ／レビューを行い、進捗・ブロッカー・リスクを早期に検出・共有します。

役割は明確に分かれており、Product Manager (PdM) は成果と優先度、Project Manager (PM) はスケジュール・リスク・コミュニケーションを担当します。開発者は実装とテスト、QA は受け入れ検証を担い、ステークホルダーは必要な承認と入力を提供します。それぞれの責任範囲はプロジェクトチャーターや役割定義に記載し、意思決定とエスカレーション経路を明確にします。

品質保証は設計段階から組み込みます。ユニットテストや統合テスト、重要フローのエンドツーエンド／スモークテストを求め、CI 上でセキュリティスキャンを実行します。リリース前には受け入れ基準の確認、CI の全チェック通過、リリースノートとロールバック計画の準備、ステージングでの検証を行い、自動化パイプラインで本番へデプロイします。問題発生時は決められたインシデント／エスカレーション手順に従って対応します。
